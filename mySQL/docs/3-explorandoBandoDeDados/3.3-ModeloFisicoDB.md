# Conhecendo o modelo físico

![modelo fisico]("../assets/insight_places_UML.png")
O modelo físico é uma representação visual de toda a estrutura do nosso sistema de gerenciamento de banco de dados, das nossas tabelas. Conseguimos ter acesso a todas as tabelas, a todas as colunas, índices, quais são as colunas interligadas, que tipo de ligação essa coluna tem com a outra coluna de outra tabela. O modelo físico é uma etapa muito importante do nosso processo de criação dos nossos bancos de dados.

Não vamos nos aprofundar nisso, porque temos uma formação de Modelagem de Dados na Alura, que explica muito bem sobre modelo físico. Ela explica a fundo nesse contexto de toda a modelagem de dados, então deixamos como indicação para você fazer essa formação.

Mas como podemos ter acesso a esse modelo físico? Através da nossa interface do MySQL Workbench, que, sim, o MySQL possibilita acesso a um modelo físico que a própria ferramenta faz. Então, é super interessante, isso é uma informação muito útil de termos. Vamos entender melhor como podemos fazer isso?

Acessando o modelo físico no MySQL
Voltando para a nossa interface do MySQL, vamos no menu da parte superior esquerda e clicamos em "Database > Reverse Engineer (Engenharia Reversa)". Também podemos usar o atalho "Ctrl + R". Assim, abrimos a janela "Reverse Engineer Database", onde mostra os parâmetros que estamos conectados no MySQL, como o hostname e username.

Na parte inferior direita da janela, clicaremos no botão "Next" (Próximo), atualizando a janela. Na nova janela, clicamos em "Next" novamente, para chegarmos na etapa "Select Schemas to Reverse Engineer" (Selecione esquemas para engenharia reversa). Então ele pede para selecionarmos qual esquema queremos fazer essa engenharia reversa, e temos uma lista com caixa de seleção.

É uma engenharia reversa porque, na verdade, é como uma etapa antes do banco de dados, para ver a modelagem do dado, que é esse modelo que precisamos selecionar. Marcaremos a opção "insight_places" e clicaremos para termos acesso a esse esquema. Depois, clicamos novamente no botão "Next" e, na etapa seguinte, clicamos no botão "Next" novamente.

Chegamos na etapa "Select Objects to Reverse Engineer" (Selecione objetos para engenharia reversa"), onde ele mostra quais objetos foram importados. Ao total, são seis tabelas selecionadas, então está correto. Na parte inferior da janela, clicaremos em "Execute" (Executar).

Na janela seguinte mostra o carregamento da execução e, quando ela finaliza, podemos clicar em "Next" novamente. Por fim, na última janela, clicamos em "Finish" (Finalizar). A janela "Reverse Engineer Database" fecha e, no MySQL, temos um modelo físico do banco de dados. Provavelmente precisamos rolar um pouco para baixo para localizar o diagrama.

Explorando o modelo físico
Se observarmos o diagrama, encontramos o desenho de todas as tabelas com todas as colunas e a identificação, por meio de setas, de quais tabelas estão interligadas. Se passarmos o mouse por cima da seta, ela vai destacar quais colunas então interligando as tabelas. Isso é muito útil para conseguirmos entender o panorama do nosso banco de dados. Isso é bem eficiente para termos essa visão do todo.

Conseguimos até fazer algumas modificações, mas não é o caso, não vamos mexer nisso. Mas, se colocarmos o mouse sobre o nome da tabela, aparece em destaque as ligações que essa tabela tem com outras tabelas, e por quais são as colunas.

Isso é muito interessante! Pode ajudar bastante a entender quando estamos trabalhando com um banco de dados enorme, com várias tabelas diferentes, e precisamos entender como vamos organizar essas consultas e quais tabelas estão se relacionando com cada tabela.

Já observamos todos os detalhes do nosso banco de dados da Insight Places. Nós descobrimos quais são as informações que a empresa armazena nessas tabelas, como essas tabelas se relacionam, quais os tipos de dados de cada coluna dessa tabela.
