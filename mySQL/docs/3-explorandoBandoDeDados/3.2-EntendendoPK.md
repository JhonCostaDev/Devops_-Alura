# Entendendo Chaves Primárias
Quais os tipos de dados são permitidos como Chave Primárias
  * Inteiros
  * UUID - Ocupa mais espaço
  * VARCHAR, CHAR
  * COMPOSTAS (UNIÃO DE MAIS DUAS CHAVES PRIMÁRIAS) - Consultas mais custosas

## Utilizando dados como chave primária
O primeiro dado é do tipo inteiro. Nós podemos usar dados do tipo número inteiro. Existe também uma função que podemos aplicar quando definimos a coluna de chave primária (ID) do tipo inteiro, que é a função de auto_increment.

Quando aplicamos essa função, fazemos com que o próprio sistema consiga gerar, sozinho, novos IDs únicos através dessa função de autoincremento, incrementando números na sequência inteira. Podemos usar tanto do tipo int como big int, que são dois tipos de valor numérico. A função de autoincremento vai criando novos IDs automaticamente, sem a necessidade de criá-los manualmente.

Outro tipo que podemos utilizar para ser chave primária dentro do MySQL é o tipo **UUID**, que é um tipo que mistura letra e número. Ele gera uma combinação de valores bem grande, então podemos usá-lo quando queremos criar tabelas com muitos dados, que talvez só números inteiros não sejam suficientes.

Com o UUID, conseguimos criar combinações entre letra e número que permitem adicionar vários registros sem se preocupar com a quantidade de valores únicos que precisamos ter na coluna de chave primária. No entanto, o UUID também ocupa mais espaço do que um número inteiro, então é um ponto negativo na utilização dele.

Outra forma de definir as chaves primárias dentro do MySQL é usando VARCHAR ou CHAR, que são tipos de texto. Podemos utilizar essas duas variações de string para também determinar as chaves primárias dentro das nossas tabelas no MySQL.

## Chaves primárias compostas
Existe ainda a possibilidade de determinar que chaves primárias compostas, ou seja, que elas sejam formadas por informações de duas ou mais colunas, que só assim elas sejam registros únicos. Elas são úteis quando a unicidade de cada valor depende de uma combinação de informações que estão em diferentes colunas. No entanto, uma desvantagem é que as consultas vão ficar um pouco mais complexas quando trabalhamos com esse tipo de chave primária.

## Question Determinando o tipo de dado de uma chave primária


Adriana está construindo seu primeiro banco de dados no MySQL, começou a estruturar as colunas de suas tabelas e, durante a escolha dos tipos de dados, percebeu que nem todos os dados podem ser utilizados na coluna classificada como chave primária da tabela, isso porque essa deve ser uma coluna de identificação dos registros e precisa ter dados de fácil acesso e comparação em suas consultas.

Quais os tipos de dados são a escolha mais recomendada para as colunas de chaves primárias das tabelas que serão criadas?


    a) As chaves primárias podem ter qualquer tipo de dado desde que sejam chaves primárias compostas, ou seja, chaves que consistem em mais de um campo que juntos formam uma chave única.


    b) Uma boa escolha seria o tipo numérico INT, que possibilita a utilização da função Auto-incremento (AUTO_INCREMENT), gerando chaves primárias únicas automaticamente. Outras opções seriam os tipos strings CHAR e VARCHAR ou o tipo UUID (Universally Unique Identifier), quando você tem a necessidade de cadastrar chaves primárias com códigos que possuem letras.


    c) Apenas os tipos de dados numéricos como INT, BIGINT, FLOAT e DECIMAL podem ser utilizados nas colunas de chaves primárias pois são os únicos que aceitam a função Auto-incremento (AUTO_INCREMENT).


    d) Apenas os tipos de dados de string como CHAR, VARCHAR e TEXT podem ser utilizados nas colunas de chaves primárias já que esses tipos possibilitam a criação de chaves primárias com letras.
